# 引导 v2:全景原地成长(替代四步向导)

> 2026-07-15 定稿。来源:一次对"引导太繁琐"的 grilling(Q1–Q8)。
> 替代上一版的"四步进度条向导"。与代码不一致时以本文为准。

## 0. 一句话

**没有独立的新手引导界面。资产全景就是唯一的界面,它自己会长大。**
新人第一次打开看到的是丰满的 demo 样板(张小满一家),每张卡带「换成我的」,
就地一格格改成自己的;现金流/保险/订阅等模块在同一个界面上逐步解锁。

## 1. grilling 的八个决定

| # | 问题 | 决定 |
|---|---|---|
| Q1 | "繁琐"的病因 | 字段太专业(持仓查询代码) + 引导 UI 噪音 + 每步字段太多 |
| Q2 | 新人第一次的门槛 | **门槛优先**:只填"资产速写"(五大类各一个约数),2 分钟见全景;实时行情/下钻/XIRR 作为填了明细才解锁的后续奖励 |
| Q3 | 速写的身份 | **永久档位**:承认有人就想粗着用,不逼他填更细。速写 = `accounts` 里的手动粗账户 |
| Q4 | 模块怎么"逐步引导" | **克制推荐**:一次只推一个建议、可静音、忽略不再提;教育用 **demo 样板展示**,不用文字科普 |
| Q5 | 成就系统 | **给予型**:庆祝"你解锁了什么能力",绝不显示"还差 X 项"。永久档位只和给予型自洽 |
| Q6 | 旧四步向导的命运 | **拆外壳留零件**:拆掉四步进度条 + 强制序列;保留目标编辑器、各模块表单、demo、成果反馈 |
| Q7 | 空全景第一眼 | **demo 填充态**:一打开就是张小满的丰满全景,每张卡「换成我的」,原地替换 |
| Q8 | 点「换成我的」后怎么填 | **分而治之**:简单卡(大类约数)就地内联秒填;复杂卡(持仓/目标/保险)滑出编辑面板(复用现有表单) |

## 2. 用户旅程

```
打开资产全景
  └ 新用户 → demo 填充态(张小满样板),顶部一条:「现在是样板 · 点「换成我的」改成你的」
        ├ 点大类色块「✎换成我的」→ 就地变输入 → 填约数 → 秒更新,样板角标消失
        │     └ 五大类都换完 → 🏆 解锁「资产全景」(第一枚成就)
        ├ 点持仓地图/其它复杂卡「换成我的→」→ 滑出编辑面板(复用 holdings_page 等)
        └ 点「还能管」里的模块「了解」→ 先看该模块的 demo 样板 → 「记我自己的」→ 填 → 🏆 解锁
  └ 老用户(已有数据)→ 直接自己的全景,无样板、无提示条
```

## 3. 拆 / 留 / 建

**拆(Q6)**:`onboard_bar` 四步进度条、`ob-step`/`ob-hint` 强制序列外壳、"第①/②/③/④步"编号叙事。

**留(Q6)**:`goal_page` 目标编辑器、各 Tab 表单、demo 样板间、`onboard_reward` 成果反馈的内核(改造为成就)。

**建**:
1. **demo 填充态全景**(Q7):新用户打开 `/` → 全景直接以 demo 数据渲染,每张卡挂「换成我的」角标 + 灰色「样板」标。
2. **大类就地内联编辑**(Q8):poster 大类色块点「✎」→ 当场变输入框 → 填 → 落地为 `accounts` 手动账户 → 全景即时重渲 → 该格「样板」标消失。
3. **样板 vs 我的 视觉区分**(Q7):未替换的卡带灰「样板」角标;替换后消失。用户一眼知道还剩哪些是张小满的。
4. **复杂卡滑出面板**(Q8):持仓/目标/保险等卡的「换成我的→」→ 滑出对应现有表单(手机=全屏 sheet)。
5. **克制推荐卡**(Q4):首页固定位置只显示一张"下一步不妨试试",可静音,忽略不再提这一个。
6. **模块 demo 教育**(Q4):未解锁模块的「了解」→ 展示该模块的 demo 样板 + "记了它你能看到什么" + 「记我自己的」。
7. **给予型成就**(Q5):每解锁一个模块 → 庆祝 toast +「已解锁」清单(往回看的骄傲);**无完成度进度条、无"还差 X 项"**。

## 4. 数据落地与坑

- **速写 = 手动粗账户**:大类"权益 30 万" → `accounts` 一行(type=权益类占位、无行情) + `manual_values` 一个值。系统原生支持(理财/存款本就是这样)。
- **去重坑(Q3 的代价)**:用户日后往某大类填**真持仓**(holdings)时,那个大类的速写粗账户会导致**重复计算**。规则:添加某大类的真持仓时,系统提醒"删掉对应速写粗账户"。一次性、局部,不是走流程。
- **demo 隔离不变**:demo 数据仍走 `storage.enable_demo()` / `demo/` 目录,`PD.demo` 标记横幅。新用户的"样板填充"复用这套。
