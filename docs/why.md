# 这套系统的「为什么」

> 操作手册见 [README](../README.md)。这一篇不讲怎么用，讲**为什么是这个形状**——
> 每一节都来自一次真实的决策或一次真实的翻车，不是理财教科书的转述。
> 你的 agent 也会读这份文档:当你问它「为什么要记现金流」时，它应该用这里的理由回答你。

---

## 一、为什么是本地文件 + agent，不是记账 App

财务数据是一个人最敏感的数据。这套系统的答案是**数据主权**:一切都在你自己的磁盘上
（sqlite + 每日明文导出双保险），没有云端、没有账号、没有第三方能看到你的净资产。

代价是没有现成的客服和引导——而这恰好是 agent 补上的。这个项目曾经认真做过一套
新人引导 UI（进度条、向导、空状态页），上线当天就被删光了:因为对着 agent 说
「这是我的情况，帮我配好」比任何向导都快。**agent 就是引导，也是编辑器**:
改目标、加保单、调定投，说一句话即可，不需要为几年一次的操作维护一套表单。

## 二、为什么先记现金流

因为**储蓄率是普通人财务里唯一完全可控的变量**。行情不可控，收入半可控，
只有「花剩多少」每个月都在你手里。

这套系统里现金流不是记账癖好，它直接驱动投资:**每月定投金额 = 真实净结余 × 80%**，
结余为负自动暂停。没有真实的现金流记录，定投金额就是拍脑袋——系统曾经用「计划结余」
跑了很久，直到对账时发现计划数和实测数差了一截:生活开销从未被记录，储蓄率其实是上限值。
先有真账，再有弹药。

现金流还有第二个身份:**增长归因的原料**。每月净资产的变化被拆成
「储蓄贡献 + 投资损益 + 房产变动」三块精确闭合——你能亲眼看到砍掉一笔订阅、
提高一次储蓄率，下个月在哪根柱子上显形。看资产不再只看结果，还能看见自己行为的效果。

## 三、为什么要立目标和 IPS 宪法

没有「为什么」的数字页面，最终都会退化成盯盘。

这个项目走过这条弯路:面板一度堆了二十多个模块、密度高到没有重点，连作者自己都说不清
「这页到底服务什么决策」。答案是从顶层补的:先写**人生目标**（几条一句话的叙事，
比如孩子培养/资产增值/家庭保障），每个目标挂上它的财务基座;再写 **IPS 投资政策声明**
——一页「宪法」，规定目标配置、容忍带、集中度红线、定投规则，以及最重要的一条:
**任何修改须经 7 天冷静期并记录理由**。

宪法的作用不是限制你，是限制**未来某个深夜想追涨杀跌的你**。

## 四、为什么配置要有目标态和容忍带

因为再平衡是纪律，不是预测。

系统用 **5/25 规则**:某类资产偏离目标 5 个百分点、或相对偏离 25%，才触发行动;
带内的一切波动都视为噪音，不动。触发后的手段也有优先级:**增量定投定向 › 分红新钱 › 卖出**
——尽量用新钱把配置「浇」回去，而不是折腾存量。

还有一个容易忽略的设计:目标配置写的是**目标态**（比如几年后卖房换房之后要长期持有的
那个组合），而不是当前状态的微调。如果你有一次量级远超日常的动作在路上（卖房、大额到账），
按当前权重优化就是在优化一条不会走的路。

## 五、为什么配保险

因为不算缺口，保险就永远是推销品;算了缺口，它变成一道工程题。

系统用**需求分析法**:寿险需求 = 全部负债 + 家庭固定支出 × N 年（默认 10 年），
减去已有保额就是缺口;重疾按成员收入的倍数单独算。这个数字算出来那天往往很刺眼——
demo 数据里的张小满一家，算出的寿险缺口就是六位数量级——但它把一个模糊的焦虑
（「我的保险够不够?」）变成了一个可执行的动作（「补一份多大保额的定期寿险」）。

原则:**先保障后收益**。保单的现金价值、保费日历、缴清年份都进台账，
续费提醒和扣费日历一起出现——保障是承诺，承诺要被管理。

## 六、为什么每一笔操作都必须写原因

因为**没有原因的交易，明年不该再出现**。

持仓台账要求每笔买卖写「原因/备注」，IPS 页会对全部历史操作跑合规审计:
违反集中度红线的、下跌期恐慌卖出的、没写原因的，都会被点名。这不是形式主义——
一年后复盘时，你只能靠这一栏区分「按计划执行」和「情绪化操作」。
写不出原因的那一刻，往往就是不该下单的那一刻。

## 七、为什么做年度复盘和人生目标问答

因为工具是**仪式的主持人，不是人生的记分员**。

每年生成一份年度报告:归因、收益、配置漂移、操作大事记自动汇总;人生目标那一节，
财务部分是活数字，非财务部分只有几个问题（今年陪孩子做成了什么?最大的一次分歧怎么解决的?）。
系统不打分、不追踪、平时一个字不提——年末由 agent 拿着问题访谈你，答案存档，
明年复盘时翻给你看。

设计边界在这里划死:财务工具能管目标的**财务基座**，管不了目标本身。
试图量化「家庭和谐度」的字段，最终都会变成自欺的摆设。

## 八、为什么有压力测试和流动性瀑布

因为它们是**为出事那天准备的**，价值不在每天。

压力测试平时收成一行体检（绿灯折叠、红灯自动展开）:权益跌三成、房价跌两成、
危机组合同时来，净资产剩多少、杠杆被动放大到多少——负债不会跟着资产缩水，
这就是杠杆的真面目。流动性瀑布则回答另一个问题:真出事了，钱按什么顺序动?
现金/货基 → 可卖资产 → 保单贷款 → 最后才是动不了的。它和平台分布一起放在
页面最底部的「应急参考区」——那是给最坏一天（也可能是给你的家人）准备的说明书。

日常你几乎不会看它们。这不是设计失败，恰恰是设计意图:
**监控系统运转正常的表现，就是大多数日子无事可报。**
